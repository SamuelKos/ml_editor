event <<BBB>> does NOT have to be binded to physical events with: event add.
Official documentation does not tell how important this is to understand.
-->INFO It is VERY important to understand that:
virtual-events doesn't have to be binded to physical events with event add.

Example:
bind $mywidget <<BBB>> {puts jou}
event generate $mywidget <<BBB>>
--> jou


Because -data attribute for virtual-events did not exist before tcl 8.5:
Must use global variables, which is not bad option actually

Important: update stringvar mydata before emitting virtual-event BBB
Otherwise procs binded to this event would get triggered with old values:
set mydata "$index $index2 $length [lindex $args 1]"
event generate $this <<BBB>>
